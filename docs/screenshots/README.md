# Screenshots

These images appear in the top-level README. Capture them against the live planner
at vivijure.skyphusion.org (studio-token gated -- use Conrad's browser session with the token pasted).

## Needed files

| Filename | What to capture |
|---|---|
| `planner-storyboard.png` | The planner page with a storyboard loaded -- scenes visible in the editor, shot descriptions filled in. Collapsed History section at bottom is fine. |
| `planner-render-history.png` | The History tab showing a mix of render rows -- one COMPLETED with a download button, one in-flight (IN_PROGRESS or IN_QUEUE), ideally one FAILED so the new inline error snippet is visible. |
| `cast-page.png` | The cast page (/cast) with at least one character loaded -- portrait visible, LoRA status shown. |
| `modules-page.png` | The modules page (/modules) showing the installed module cards (keyframe, own-gpu, etc.). |

## Capture tips

- 1400px wide viewport gives a clean layout without squeezing or over-stretching.
- Collapse the browser chrome (full-screen or minimal toolbar) before screenshotting.
- The planner storyboard shot is the hero image -- make it look good (a real scene with
  shot descriptions, not placeholder text).
- PNG, no compression artifacts.

Drop the files here and the README will pick them up automatically.
